# PyMARS Installation Guide (CPU and GPU)

This guide explains how to install PyMARS and FeNNol on shared HPC clusters and Linux systems.
The instructions are designed to:
* avoid system-wide installations
* not require root/admin access
* minimize native compilation problems
* improve compatibility with older cluster environments
Needed Python version:
* Python 3.10

======================================================================
SECTION 1 — CPU INSTALLATION
======================================================================
## Main Installation Procedure:
programs needed:
* conda
* Python
* pip
* git
* uv (if needed, simply do pip install uv)

Please ensure these are installed beforehand.

1. Create a virtual environment
   conda create -n pymars-env python=3.10 -y
    
2. Activate the environment:
   conda activate ~/.conda/envs/pymars-env

3. Upgrade build tools:
   pip install --upgrade pip setuptools wheel

4. Install CPU JAX:
   pip install --upgrade jax

5. Fetch PyMARS (either clone or go to the program directory):
   git clone https://github.com/thomasple/pymars.git pymars_dir

   cd pymars_dir

6. Generate/update the uv lockfile:

   uv lock --upgrade

7. Install dependencies from the project TOML file:

   pip install .
If editable mode is preferred:
```
pip install -e . --no-deps
```
If there are issues, try to install with poetry (dependencies solver):
```
pip install poetry
poetry install
```

8. Install remaining dependencies manually:
```
pip install \
    h5py \
    ase \
    numba \
    optax \
    sympy \
    tomlkit \
    pytest \
    flax \
    msgpack \
    rich \
    treescope \
    typing_extensions \
    pyyaml
```
10. Install FeNNol:
```
pip install fennol --no-deps
```
11. Test the installation:
```
python -c "import jax; print('JAX version:', jax.__version__)"
python -c "import jax; print('Devices:', jax.devices())"
python -c "from fennol import FENNIX; print('FeNNol OK')"
python -c "import pymars; print('PyMARS OK')"
pymars --help
```

## Alternative Installation (micromamba) (proposed)
If the pip-only installation fails because of native dependencies, micromamba is usually the best alternative on clusters.
1. Install micromamba locally (no admin privileges required):

   curl -L micro.mamba.pm/install.sh | bash
2. Restart the shell or source your shell configuration.
3. Create the environment:

   micromamba create -n pymars python=3.10 -y
4. Activate the environment:

   micromamba activate pymars
5. Install heavy native packages:

   micromamba install -c conda-forge 
   h5py 
   tensorstore 
   hdf5 
   ase 
   -y
6. Install uv:

   pip install uv
7. Install JAX CPU:

   pip install --upgrade jax
8. Clone PyMARS:

   git clone https://github.com/thomasple/pymars.git pymars_new

   cd pymars_new
9. Generate/update the lockfile:

   uv lock --upgrade
10. Install from TOML:

    pip install .
11. Install FeNNol:

    pip install fennol --no-deps
12. Install remaining Python dependencies:

    pip install 
    numba 
    optax 
    sympy 
    tomlkit 
    pytest 
    flax 
    msgpack 
    rich 
    treescope 
    typing_extensions 
    pyyaml
13. Test the installation:

    python -c "import jax; print(jax.devices())"

## Common Problems

On older nodes, it might be useful to load the most recent GCC module available.

1. h5py build failure
Typical error:
```
libhdf5.so: cannot open shared object file
```
Cause:
* pip attempted local compilation
* HDF5 development libraries are unavailable
Fixes:
* avoid local compilation
* prefer precompiled wheels

2. tensorstore build failure

Typical errors:
```
GLIBC_2.25 not found
GLIBCXX_* not found
CXXABI_* not found
```
Cause:
* cluster operating system too old
* incompatible compiled binaries
Fixes:
* avoid optional tensorstore/orbax dependencies
* avoid flax[all]
* install lightweight versions without dependencies:

  pip install flax --no-deps
  pip install optax --no-deps
  pip install rich --no-deps

Ensure pyyaml is installed.
Then try to run the program: if absl is missing, but flax, optax and rich are present, clear cache (pip cache purge)
 and install optax and rich with dependencies (absl is a dependency of these).

------Troubleshooting deps useful commands to try: (flax, optax, rich)
	# Install h5py and other problematic packages from conda-forge
	conda install -c conda-forge h5py tensorstore -y

	# Skip FeNNol and just install pymars with minimal dependencies
	pip install -e . --no-deps
	pip install pyyaml ase numpy scipy

	# Install flax WITHOUT the [all] extras (avoids tensorstore rebuild)
	pip install flax --no-deps
	pip install msgpack rich treescope typing_extensions

	# Now install fennol without dependencies
	pip install fennol --no-deps

	# Install remaining fennol dependencies manually
	pip install numba sympy tomlkit pytest
	pip install optax PyYAML

	# Finally install pymars
	cd ~/pymars_new
	pip install -e . --no-deps
	pip install pyyaml ase

* alternatively use micromamba packages
3. AVX runtime errors

Cause:
* login nodes may use older CPUs incompatible with JAX wheels

Fix:
* run on compute nodes instead

4. Disk space issues
Fix:
```
pip cache purge
```
Remove unused virtual environments if necessary.
## Troubleshooting
Minimal fallback installation:
```
python3 -m venv pymars-env
source pymars-env/bin/activate
pip install --upgrade pip setuptools wheel
pip install uv
pip install jax
git clone https://github.com/thomasple/pymars.git pymars_new
cd pymars_new
uv lock --upgrade
pip install -e . --no-deps
pip install \
    flax --no-deps \
    optax --no-deps \
    rich --no-deps
pip install \
    pyyaml \
    numba \
    sympy \
    msgpack \
    ase

pip install fennol --no-deps
```
If installation still fails:
* avoid optional extras
* avoid tensorstore/orbax
* use micromamba instead of pure pip
* run on compute nodes rather than login nodes

Work around failing dependencies by installing with no deps, don't try to update tensorstore, h5py or hdf5 if it fails the first time
(most likely it will be a waste of time).


======================================================================
SECTION 2 — GPU INSTALLATION
======================================================================

## Main Installation

This installation assumes:

* NVIDIA GPUs are available
* CUDA drivers are already installed by the cluster administrators
* the user has no admin privileges

1. Create a virtual environment:

   python3.10 -m venv pymars-gpu

2. Activate the environment:

   source pymars-gpu/bin/activate

3. Upgrade build tools:

   pip install --upgrade pip setuptools wheel

4. Install uv:

   pip install uv

5. Verify GPU visibility:

   nvidia-smi

6. Install CUDA-enabled JAX

Check CUDA version first:
```
nvidia-smi
```

For CUDA 12:
```
pip install --upgrade "jax[cuda12]"
```

For CUDA 11:
```
pip install --upgrade "jax[cuda11]"
```

Alternative installation:
```
python -m pip install \
  --upgrade \
  "jax[cuda12]" \
  -f https://storage.googleapis.com/jax-releases/jax_cuda_releases.html
```

7. Clone PyMARS:

```
git clone https://github.com/thomasple/pymars.git pymars_new
cd pymars_new
```

8. Generate/update the uv lockfile:
```
uv lock --upgrade
```

9. Install from TOML:
```
pip install .
```

If editable mode is preferred:
```
pip install -e . --no-deps
```

10. Install remaining dependencies:
```
pip install \
    h5py \
    ase \
    numba \
    optax \
    sympy \
    tomlkit \
    pytest \
    flax \
    msgpack \
    rich \
    treescope \
    typing_extensions \
    pyyaml
```

11. Install FeNNol:
```
pip install fennol --no-deps
```

12. Test GPU visibility:

```
python -c "import jax; print(jax.devices())"
```

Expected output:
```
[GpuDevice(id=0)]
```

## Common Problems

1. GPU not detected

Typical output:

```
Devices: [CpuDevice(id=0)]
```

Fix:

```
python -m pip uninstall -y jax jaxlib
```

Then reinstall CUDA-enabled JAX.

2. CUDA mismatch

Cause:

* installed JAX wheel incompatible with CUDA runtime

Fix:

* verify CUDA version with:

  nvidia-smi

* reinstall matching JAX wheel

3. GLIBC / tensorstore / flax errors

Typical errors:

```
GLIBC_2.25 not found
GLIBCXX_* not found
```

Fix:

```
pip install flax --no-deps
pip install optax --no-deps
pip install rich --no-deps
```

Avoid:

* flax[all]
* orbax
* tensorstore-heavy extras

4. AVX runtime errors

Cause:

* old CPUs on login nodes

Fix:

* use compute/GPU nodes only

## Troubleshooting

Force CUDA backend:

```
export JAX_PLATFORMS=cuda
```

Select GPU manually:

```
export MARS_DEVICE=cuda:0
```

Force CPU mode:

```
export MARS_DEVICE=cpu
```

Minimal fallback installation:

```
python3 -m venv pymars-gpu

source pymars-gpu/bin/activate

pip install --upgrade pip setuptools wheel

pip install uv

pip install --upgrade "jax[cuda12]"

git clone https://github.com/thomasple/pymars.git pymars_new

cd pymars_new

uv lock --upgrade

pip install -e . --no-deps

pip install \
    flax --no-deps \
    optax --no-deps \
    rich --no-deps

pip install \
    pyyaml \
    numba \
    sympy \
    msgpack \
    ase

pip install fennol --no-deps
```

Test:

```
python -c "import jax; print(jax.devices())"
```

Expected output:

```
[GpuDevice(id=0)]
```

## Additional Notes

Device selection in input.yaml:

```
calculation_parameters:
  device: cuda:0
```

Or using environment variables:

```
export MARS_DEVICE=cuda:0
```

To remove environments:

For venv:

```
rm -rf pymars-env
```

For micromamba:

```
micromamba env remove -n pymars
```
