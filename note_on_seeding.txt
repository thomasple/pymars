1) Where the seed is parsed and stored
File: __init__.py
Location: The “Set per‑trajectory seeds” block in main() .

Key facts from that block:

The user‑facing key is still general_parameters.seed.
If batch_size > 1 and seed is:
list/tuple/ndarray → treated as the per‑trajectory seed list.
comma‑separated string like "1,2,3" → split and treated as per‑trajectory seed list.
If batch_size == 1 and seed is scalar → used directly.
If batch_size > 1 and seed is scalar/non‑list or missing → per‑trajectory random seeds are generated; a warning is printed if scalar.
Uniqueness enforced for batch runs (duplicates in user list raise an error; duplicates in random list are regenerated).
Finally, the resolved per‑trajectory seed list is passed to MD initialization:


simulation_parameters["general_parameters"]["seed_list"] = trajectory_seeds
So the per‑trajectory seeds used for stochastic initialization are always stored under general_parameters.seed_list.

3) Random rotation
File: md.py
Section: “random rotation”


coordinates = np.stack(    [apply_random_rotation(coordinates, n_rotations=None, rng=trajectory_rngs[b])     for b in range(batch_size)],    axis=0,)
So each trajectory uses its own RNG. The RNG is passed into:

File: rotation_utils.py
Function: apply_random_rotation


M = R.random(nrot, random_state=rng).as_matrix()
Key fact:

scipy.spatial.transform.Rotation.random(..., random_state=rng) is seeded by the per‑trajectory RNG passed in from md.py.
✅ Therefore, random rotation is fully reproducible per trajectory, based on the seed in seed_list[b].

5) Summary (seed → rotation + velocity)
general_parameters.seed → parsed in __init__.py → stored as seed_list
seed_list[b] → np.random.default_rng(seed_list[b])
That RNG is used for:
random rotation (via apply_random_rotation(..., rng=trajectory_rngs[b]))
initial velocities (via sample_velocities(..., rng=trajectory_rngs[b]))
So the seed directly controls both the random rotation and the velocity initialization per trajectory.
